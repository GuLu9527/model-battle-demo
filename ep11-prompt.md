# EP11 题目 Prompt · 多足生物群仿真

> **场景**：录制时直接复制粘贴到 7 个 AI 客户端（同份 prompt + 60 分钟限时）
> **题目类型**：单文件 HTML / Canvas2D 视觉同题对决
> **难度定位**：IK + 状态机 + 群体行为，比 EP01-08 视觉题难一档（限时从 30 分钟提至 60 分钟）

---

## 主 Prompt（直接复制给 7 个模型）

```
做一个单文件 HTML 的多足生物群仿真，要求：

【硬约束】
1. 单个 HTML 文件，零外部依赖
   - 不允许 CDN、不允许 import、不允许任何动画库
   - 不允许预录的关键帧 / mocap 数据 / 外部图片 / 外部 SVG
2. 双击 HTML 文件就能在浏览器跑（offline 可用）
3. 渲染只能用 Canvas 2D，禁用 WebGL
4. 单线程实现，不要 Web Worker / OffscreenCanvas
5. IK 解算函数必须保留可读结构（FABRIK / CCD / two-bone analytic 都行）
6. 文件大小不超过 250KB

【生物形态 — 至少实现两种】
1. 多足生物：4 腿或 6 腿
   - 每条腿是一条 IK 链，至少 2 段（大腿 + 小腿，可加足）
   - 步态有「支撑相」和「摆动相」交替
   - **支撑相脚必须锚定不打滑**（脚在地面那一刻起，世界坐标固定，直到抬起）
   - 摆动相脚抬起，沿弧线落到下一个目标点
2. 爬虫：分节式身体
   - 头部跟随移动方向
   - 后续节段用「距离约束」依次传播（前一段动了，后一段才被拉过来）
   - 至少 6 个节段

【运动 / 控制】
1. 鼠标点击或按键切换两种模式：
   - 模式 A「跟鼠标」：所有生物追逐鼠标位置
   - 模式 B「自主游荡」：每只生物有自己的目标点 / wander 行为
2. 二维地形（任意高度图）
   - 程序生成的连续地形（柏林噪声 / sin 叠加都行）
   - 至少有起伏（不能是平地）
3. 脚必须落在地面上（用地形高度图查询）
4. 身体高度随地形起伏
   - 多足生物：身体高度 = 当前所有支撑脚平均高度 + 偏移
   - 爬虫：每节都贴着地表

【性能硬指标】
- 同窗口 5 只生物，稳定 60fps（用 stats 显示帧率）
- 帧率不达标直接扣性能分

【UI / 视觉】
- 2D 侧视图（X 横向 + Y 高度），不要 3D 透视
  - 一旦做成 3D 视图（哪怕假 3D / 等距投影），性能 + 视觉两项都不给分
- 顶部 HUD：FPS / 当前模式 / 生物数 / 简短说明
- 控制按钮：「+ 生物」「- 生物」「切换模式」「重置地形」
- 整体观感：不要纯线框，至少有填充 + 阴影 + 地形纹理

【调试可见性】
- 提供一个 debug 开关（按 D 键）：
  - 显示 IK 链骨骼线
  - 显示脚目标点
  - 显示支撑相 / 摆动相状态色（红 / 蓝）
  - 显示步态相位条

【验收清单（你交付前自己跑一遍）】
□ 双击 HTML 就能跑（不依赖任何外部资源）
□ 至少 5 只生物同屏 60fps
□ 4/6 腿生物：支撑脚不打滑（盯一只腿看 3 秒，脚的世界坐标不动）
□ 爬虫：身体节段距离稳定
□ 地形不是平地，脚跟随地形高度
□ 模式 A、模式 B 都能切换且工作
□ Debug 模式可开（D 键）

【代码组织建议】
- 顶部一段 const 配置区（颜色、腿长、步频、群体规模）
- IK 解算函数独立可读
- 主循环：update 物理 → IK 解算 → 绘制
- 注释用中文，关键魔数标注含义
```

## 评分维度（评审用，不告诉模型）

| 维度 | 满分 | 关键评分点 |
|---|---|---|
| **IK 数学** | 100 | 链段长度守恒（≥99%）/ 解算稳定（无抖动）/ 链段结构清晰可读 / 边界处理 |
| **状态机** | 100 | 支撑/摆动相切换 / 速度→步频映射 / 多腿相位差 / 模式切换无 bug |
| **二次运动** | 100 | 身体随步态垂直起伏 / 反向脊柱 / 头部稳定 / 尾巴/触手摆动 |
| **群体行为** | 100 | 至少分离/对齐/聚合 1 项；3 项全有 + 90 分；做到捕食者+逃跑步频提升 ≥95 分 |
| **性能** | 100 | 5 只 60fps = 80；10 只 30fps+ = +10；20 只仍 30fps+ = +10 |
| **视觉调优** | 100 | 阴影 / 地形纹理 / 颜色协调 / 整体观感 / 手感 |

加权综合 = `(IK×1.0 + 状态机×0.9 + 二次运动×0.8 + 群体×1.1 + 性能×1.0 + 视觉×0.7) / 5.5`

## 加分项（不入提示词，评分时单独标注）

| 项 | 说明 | 加分 |
|---|---|---|
| 步态相位差正确 | 4 腿对角步态 / 6 腿三角步态合理 | +5 |
| 身体二次运动 | 身体随步态有规律起伏 + 反向脊柱 | +5 |
| 群体避让 | boids 三原则完整实现 | +10 |
| 落脚预测 | 落脚点用速度外推（不是当前位置投影） | +5 |
| 捕食者-猎物 | 一种追、一种逃、逃跑步频提升 | +10 |
| 外观可变 | 随机 seed 生成不同体型且仍正常行走 | +10 |

满分外加 +45。我们公开"满分 100 + 加分 45 = 上限 145"，让观众明白冠军真正的差距。

## Prompt 设计原理

### 为什么这道题适合 EP11

| 维度 | EP09 魔方（抖音翻车） | EP10 真PR（抖音翻车） | EP11 多足生物（预期回归） |
|---|---|---|---|
| 视觉冲击 | 中（魔方静止） | 弱（代码 IDE） | **强**（生物在跑） |
| 即时奖励 | 中（求解动画） | 低（要看 diff） | **极高**（一眼判断打滑/穿模/僵硬）|
| 同屏对比 | 弱 | 弱 | **极强**（7 模型 5 只生物 = 35 个生物同屏对比）|
| 翻车视觉化 | 弱 | 弱 | **极强**（腿穿地、瘸腿、打滑、抽搐都是 meme 素材）|

### 防作弊设计

| 风险 | 防御 |
|---|---|
| 用预录关键帧伪装 IK | 提示词明禁；评分时盯支撑相判定 |
| 用 mocap 数据 | 提示词明禁；评分时检查 IK 函数是否真求解 |
| 拖外部 SVG / 贴图 | 提示词明禁；单文件 + 浏览器 offline 验证 |
| 假 60fps（跳帧） | 评分时强制 stats.js 显示帧率 + 录屏看 |
| 3D 视图蒙混 | 提示词明确 2D 侧视图，3D 直接 0 分 |

### 为什么加分项不入提示词

加分项进 prompt 会让模型功能堆砌、忽略硬指标。**让模型自由 60 分钟，主动做加分项的，才是真有功力**。这也是 EP07/EP08 同题对决产生差异化的关键。

### 60 分钟限时（vs EP09 的 30 分钟）

EP09 30 分钟够手写 3D + 求解器吗？— 1/9 完成（DeepSeek 32D 显示）。教训：**真求解算法 + 视觉渲染同时做，30 分钟太紧**。

EP11 同时要：IK 实现 + 步态状态机 + 地形生成 + 群体逻辑 + 性能优化 — 比 EP09 重一档。
- 30 分钟：可能没人能跑完
- 60 分钟：阵容全员都有机会做出可演示版本，差异化体现在加分项
