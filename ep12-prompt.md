# EP12 题目 Prompt · 弹珠漏斗耐力赛

> **场景**：录制时直接复制粘贴到 8 个 AI 客户端
> **题目类型**：弹珠漏斗耐力赛（Funnel Endurance）— 复刻 Jelle's Marble Runs 经典赛事
> **难度定位**：物理仿真 + 碰撞系统 + 赛制逻辑，视觉驱动型题目

---

## 主 Prompt（直接复制给 8 个模型）

```
做一个弹珠漏斗耐力赛（Funnel Endurance）模拟器。

灵感来自 Jelle's Marble Runs (JMR) 的经典漏斗赛项目。目标是复刻真实比赛的视觉效果和物理行为。

【赛道设计】
1. 至少 5 层漏斗垂直串联
   - 每层漏斗是一个倒锥形，底部中心有下落孔
   - 漏斗大小可以不同（上小下大，或交替变化）
   - 漏斗颜色/外观可以不同（模仿 JMR 的 Hubelino 橙色、MaBoRun 蓝/绿风格）
   - 漏斗壁有物理碰撞属性（弹珠碰到会反弹、摩擦减速）

2. 漏斗剖面视角（能看到弹珠在漏斗内运动）
   - 可以是正面视角（漏斗切面）或轻微 3D 透视
   - 关键是观众能清楚看到每颗弹珠的位置和运动轨迹

【弹珠】
1. 至少 8 颗弹珠同时从顶部释放
2. 每颗弹珠外观必须不同：
   - 不同颜色
   - 不同花纹（条纹/纯色/渐变/斑点等）
   - 可选：队名标签、logo、编号
   - 让观众能一眼区分每颗弹珠
3. 弹珠有阴影/高光/旋转效果，看起来像真实弹珠

【物理引擎】
1. 重力：弹珠受重力影响下落
2. 漏斗壁碰撞：弹珠碰到漏斗壁会反弹，沿漏斗壁做圆周/螺旋运动
3. 摩擦/阻尼：弹珠在漏斗壁上运动时能量逐渐损耗，轨道半径自然收缩
4. 弹珠间碰撞：弹珠之间发生弹性碰撞
   - 这会产生「驱逐效应」：一颗快速下落的弹珠撞到正在边缘滑行的弹珠，后者获得能量重新弹回外圈「续命」，前者则减速掉入下层
5. 最终弹珠通过每层漏斗的中心孔掉入下一层

【赛制 — 反向淘汰】
1. 目标：最慢掉出最后一层漏斗的弹珠获胜
2. 计时器：从弹珠释放开始计时，每颗弹珠掉出最后一层时记录时间
3. 状态显示：
   - 每颗弹珠当前在第几层
   - 已淘汰的弹珠和淘汰时间
   - 当前存活数
4. 淘汰动画：弹珠掉出时有视觉提示（变灰/爆炸/标记）
5. 胜利动画：最后一颗弹珠掉出时有庆祝效果

【交互】
1. 开始/重置按钮
2. 可选：暂停/慢放
3. 可选：点击添加弹珠

【视觉要求】
1. 漏斗有材质感（不是纯色平面，有纹理/渐变/光泽）
2. 弹珠有运动轨迹/尾迹（可选但加分）
3. 整体画面有赛道感（背景可以有简单装饰）
4. 弹珠在漏斗壁上旋转时有视觉反馈（自转动画）

【验收清单】
□ 弹珠能正常释放并沿漏斗壁螺旋运动
□ 能量损耗导致轨道收缩，弹珠最终掉入中心孔
□ 弹珠间碰撞会产生驱逐/续命效果
□ 每颗弹珠外观不同，能一眼区分
□ 反向淘汰赛制正确运行
□ 8+ 颗弹珠同时运行流畅
□ 视觉效果有 JMR 比赛的感觉
```

## 评分维度（评审用，不告诉模型）

| 维度          | 满分  | 关键评分点                    |
| ----------- | --- | ------------------------ |
| **物理真实感**   | 100 | 重力/碰撞/摩擦/能量衰减/轨道收缩/弹珠间交互 |
| **漏斗赛道**    | 100 | 多层设计/材质差异/孔径变化/视觉层次      |
| **弹珠辨识度**   | 100 | 每颗外观不同/队名/花纹/旋转动画/阴影高光   |
| **赛制还原**    | 100 | 反向淘汰/计时/淘汰动画/状态追踪/JMR 规则 |
| **JMR 还原度** | 100 | 整体氛围/赛道装饰/戏剧性/观众感/边缘行者效果 |
| **性能+稳定**   | 100 | 8+弹珠流畅/不崩溃/长时间运行稳定       |

加权综合 = `(物理×1.2 + 漏斗×1.0 + 辨识度×1.1 + 赛制×0.8 + JMR×1.0 + 性能×0.9) / 6.0`

## Prompt 设计原理

### 为什么这道题适合 EP12

| 维度   | EP11 多足生物    | EP12 弹珠漏斗            |
| ---- | ------------ | -------------------- |
| 视觉冲击 | 中（IK/群仿真门槛高） | **极强**（弹珠碰撞+掉落，0 门槛） |
| 即时奖励 | 中（要看懂 IK）    | **极高**（一眼看出谁快谁慢）     |
| 同屏对比 | 强（35 个生物）    | **极强**（8 颗弹珠同屏竞技）    |
| 悬念感  | 弱（结果不直观）     | **极强**（谁最后掉？反向淘汰）    |
| 观众门槛 | 高（要懂物理/IK）   | **低**（弹珠+漏斗=常识）      |

### 为什么不限技术栈

EP11 限制 Canvas2D + 单文件 HTML 导致部分模型受限。EP12 不限技术栈，让模型自由发挥，核心评判标准是**谁最像 JMR 真实比赛**。

### 防作弊设计

| 风险           | 防御            |
| ------------ | ------------- |
| 预录动画伪装物理     | 观察弹珠间碰撞是否真实交互 |
| 假碰撞（弹珠穿过彼此）  | 检查碰撞检测是否生效    |
| 魔法续命（人为延长某颗） | 检查物理参数是否统一    |
| 性能作弊（减少弹珠数）  | 评分时强制加到 8+ 颗  |
